1. FTP Server (vsftpd)
Purpose: Transfer files between client & server.

Commands:

bash
Copy
Edit
# Install FTP server
sudo apt update
sudo apt install vsftpd -y

# Start & enable service
sudo systemctl start vsftpd
sudo systemctl enable vsftpd

# Edit config (optional for anonymous/secured access)
sudo nano /etc/vsftpd.conf

# Restart service
sudo systemctl restart vsftpd

# Test from client
ftp <server_ip>
Explanation: vsftpd = Very Secure FTP Daemon; allows file transfers using FTP protocol.

2. Telnet Server
Purpose: Remote login into another system using command line.

Commands:

bash
Copy
Edit
# Install telnet server
sudo apt install telnetd -y

# Start & enable service
sudo systemctl start inetd
sudo systemctl enable inetd

# On client machine
telnet <server_ip>
Explanation: Telnet is insecure (plaintext), but used for basic remote access in labs.

3. Inbuilt Firewall (UFW)
Purpose: Control network traffic to/from server.

Commands:

bash
Copy
Edit
# Install (if needed)
sudo apt install ufw -y

# Enable firewall
sudo ufw enable

# Allow port 21 (FTP)
sudo ufw allow 21/tcp

# Allow port 23 (Telnet)
sudo ufw allow 23/tcp

# Check status
sudo ufw status
Explanation: UFW = Uncomplicated Firewall; simple interface for managing iptables.

5. Proxy Server (Squid)
Purpose: Act as middleman for internet requests, can cache and filter.

Commands:

bash
Copy
Edit
# Install squid
sudo apt install squid -y

# Backup config
sudo cp /etc/squid/squid.conf /etc/squid/squid.conf.bak

# Edit config
sudo nano /etc/squid/squid.conf
# Change http_port 3128
# Set allowed networks (acl localnet)

# Restart squid
sudo systemctl restart squid

# Test with browser proxy settings or curl
curl --proxy http://<server_ip>:3128 http://example.com
Explanation: Squid caches and filters HTTP requests to improve speed/security.

6. Samba Server
Purpose: Share files/folders between Linux and Windows.

Commands:

bash
Copy
Edit
sudo apt install samba -y
sudo nano /etc/samba/smb.conf
# Add:
[shared]
   path = /home/user/shared
   read only = no
   guest ok = yes
sudo systemctl restart smbd
Explanation: Samba lets Linux share folders to Windows as a network share.

7. DHCP Server
Purpose: Automatically assign IP addresses to clients.

Commands:

bash
Copy
Edit
sudo apt install isc-dhcp-server -y
sudo nano /etc/dhcp/dhcpd.conf
# Example:
subnet 192.168.1.0 netmask 255.255.255.0 {
  range 192.168.1.50 192.168.1.100;
  option routers 192.168.1.1;
}
sudo systemctl restart isc-dhcp-server
Explanation: DHCP = Dynamic Host Configuration Protocol.

Edit

sudo systemctl restart postfix
Explanation: Postfix handles sending email from server to other mail
